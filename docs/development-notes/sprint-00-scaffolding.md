# Sprint-0 Development Notes --- Scaffolding

**Close date:** 2026-04-21
**Close time (local):** 05:58 CDT
**Close commit:** `2bdfa42`

## What I Built

A monorepo scaffold. No runtime. Directory tree, empty schemas, README
placeholders declaring status per directory.

## Defects Encountered

Two defects, both discovered during Sprint-0 execution and both remediated in
Sprint-1:

### Defect S0.D1 --- Schema extraction pollution

During v1 schema extraction from the demo's dataclasses, I generated
diagnostic dumps as reference material:

    shared/artifact-contracts/v2/_v2_schemas_dump.txt
    shared/artifact-contracts/v2/_v2_schemas_dump_2.txt
    shared/artifact-contracts/v1/_v1_schemas_dump.txt

One Office lock file was left behind:

    shared/artifact-contracts/v2/~$2_schemas_dump.txt

These were working artifacts, not contracts. I did not `.gitignore` them in
Sprint-0, which meant they showed up in `git status` across Sprint-1
planning. Remediation landed in Sprint-1 via:

    # Transient schema dump and Office lock artifacts
    **/shared/artifact-contracts/**/_*_dump*.txt
    **/shared/artifact-contracts/**/~$*

### Defect S0.D2 --- Demo ledger state untracked

The `/demo/ledger/` directory contained test-run state generated by the
demo's own test suite. This is runtime state, not governance artifact, and
should never be committed. I did not catch this until Sprint-1 `git status`
showed `demo/ledger/` as untracked. The resolution was declaring `/demo` out
of scope for Sprint-1 and accepting `demo/ledger/` as permanently untracked.

Documented in `/docs/release-notes/sprint-0-reconciliation.md`.

## Decisions I Made

I extracted v1 schemas post-hoc rather than treating them as undeclared
implementation detail. The alternative was easier --- the demo works without
formal schemas --- but would have left v2 inheriting from implementation
rather than from a versioned contract. Given that Invariant 4 requires the
ledger to be self-describing, and the v2 extension would share the parent
contract, skipping the extraction would have created a contract-versioning
problem later that would be more expensive to fix than doing the extraction
now. I extracted.

I declared v1 locked. The alternative was to allow the v1 schemas to evolve
alongside v2. I rejected this because the research narrative depends on v1
being the defended artifact. Any drift between the schemas under
`/shared/artifact-contracts/v1/` and the implementation under `/demo/`
weakens the defensibility claim.

## What I Would Do Differently

Two things.

First, I should have established `.gitignore` discipline before starting
schema extraction. The diagnostic dumps were foreseeable. Starting with the
`.gitignore` would have meant Sprint-0 closed cleanly without Sprint-1
remediation work.

Second, I should have scanned `/demo/` for untracked runtime state before
declaring Sprint-0 complete. `demo/ledger/` was predictable --- any demo with
a test suite generates ledger state --- and should have been surfaced in
Sprint-0 rather than Sprint-1.
